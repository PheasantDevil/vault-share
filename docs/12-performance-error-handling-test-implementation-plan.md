## 3. データ設計・状態管理

### 3.1 ページネーション

```typescript
interface PaginationParams {
  limit: number;      // 1ページあたりの件数（デフォルト: 20）
  offset: number;     // オフセット（デフォルト: 0）
  searchQuery?: string;  // 検索クエリ
  typeFilter?: string;   // タイプフィルタ
}

interface PaginationResponse<T> {
  items: T[];
  total: number;
  limit: number;
  offset: number;
  hasMore: boolean;
}
```

### 3.2 エラーレスポンス

```typescript
interface ErrorResponse {
  error: {
    code: string;        // エラーコード（例: 'VALIDATION_ERROR', 'NOT_FOUND'）
    message: string;      // エラーメッセージ
    details?: Record<string, unknown>;  // 詳細情報
  };
}
```

### 3.3 トースト通知

```typescript
interface Toast {
  id: string;
  type: 'success' | 'error' | 'warning' | 'info';
  message: string;
  duration?: number;  // ミリ秒（デフォルト: 3000）
}
```

## 6. 実装詳細

### 6.1 ページネーションAPI

**エンドポイント**: `GET /api/groups/[id]/items?limit=20&offset=0&search=query&type=password`

**実装内容**:
- Firestoreクエリに`limit()`と`offset()`を追加
- 検索クエリがある場合は、タイトルでフィルタリング（部分一致）
- 総件数の取得（`count()`または別クエリ）

### 6.2 トースト通知

**実装内容**:
- `ToastProvider`コンテキストの作成
- `useToast`フックの実装
- `Toast`コンポーネントの実装（アニメーション付き）

### 6.3 テスト

**実装内容**:
- Vitestを使用したユニットテスト
- Playwrightを使用したE2Eテスト
- CI/CDでのテスト実行設定

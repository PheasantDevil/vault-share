## 次の実装候補

### 1. 統合テスト・E2Eテストの追加 — 高優先度

**概要:**
- 現在はユニットテストのみ。APIエンドポイントやユーザーフローの統合テスト・E2Eテストが必要

**実装内容:**

1. **統合テストの追加**
   - APIエンドポイントの統合テスト（認証、グループCRUD、アイテムCRUD）
   - データベース操作の統合テスト
   - 認証フローの統合テスト

2. **E2Eテストの追加**
   - Playwrightを使用したE2Eテスト
   - 主要なユーザーフローのテスト：
     - ログイン → グループ作成 → アイテム作成 → 削除
     - 招待フロー（招待発行 → 招待受け入れ）
     - MFA登録・ログインフロー
     - 1Passwordインポートフロー

**懸念事項:**
- テスト環境のセットアップ（Firestoreエミュレータ、認証モック等）
- テストのメンテナンスコスト
- CI/CDでのテスト実行時間

**参考:**
- Playwright: [Playwright Documentation](https://playwright.dev/)
- Firestore Emulator: [Firestore Emulator](https://firebase.google.com/docs/emulator-suite)

---

### 2. 監査ログの拡充・可視化 — 中優先度

**概要:**
- 現在は基本的な監査ログのみ。より詳細なログと可視化機能が必要

**実装内容:**

1. **監査ログの拡充**
   - より詳細な操作ログ（IPアドレス、ユーザーエージェント、タイムスタンプ等）
   - エラー発生時のログ記録
   - セキュリティイベントの記録（不正アクセス試行等）

2. **監査ログの可視化**
   - 監査ログ一覧ページの追加（`/dashboard/audit-logs`）
   - フィルタリング機能（日付範囲、ユーザー、操作種別、グループ等）
   - ソート機能（日時順、ユーザー順等）
   - エクスポート機能（CSV形式）
   - ページネーション対応

**懸念事項:**
- Firestoreの無料枠を超えないようにする（読み取り5万/日）
- ログの保持期間の設定（30日、90日、1年等）
- 大量ログのパフォーマンス

**参考:**
- Firestoreのクエリ最適化: [Firestore Query Optimization](https://cloud.google.com/firestore/docs/query-data/queries)

**概要:**

2. **アクセシビリティの向上**
   - ARIA属性の追加・改善
   - キーボードナビゲーションの改善
   - スクリーンリーダー対応の改善
   - コントラスト比の改善
   - フォーカス表示の改善

**懸念事項:**
- デザインの一貫性
- ブラウザ互換性
- テスト方法（アクセシビリティテストツール）

**参考:**
- WCAG 2.1: [Web Content Accessibility Guidelines](https://www.w3.org/WAI/WCAG21/quickref/)
- Next.js Accessibility: [Next.js Accessibility](https://nextjs.org/docs/app/building-your-application/accessibility)

### 4. セキュリティ強化 — 低優先度

**概要:**
- 基本的なセキュリティ対策は実装済みだが、さらなる強化が可能

**実装内容:**

1. **セキュリティヘッダの設定**
   - CSP（Content Security Policy）の設定
   - HSTSの設定
   - X-Frame-Options、X-Content-Type-Options等の設定
   - Next.jsの`next.config.js`で設定

2. **レート制限の実装**
   - APIエンドポイントへのレート制限（IPアドレスベース）
   - ログイン試行回数の制限（ブルートフォース対策）
   - インポート/エクスポートのレート制限
   - FirestoreまたはRedisを使用したレート制限実装

3. **セッション管理の改善**
   - セッションタイムアウトの設定（30分、1時間等）
   - セッションの無効化機能（全デバイスからログアウト等）
   - セッション一覧の表示（アクティブセッション管理）

**懸念事項:**
- レート制限の適切な設定値
- ユーザー体験への影響
- セッション管理の複雑さ

**参考:**
- Next.js Security Headers: [Next.js Security Headers](https://nextjs.org/docs/app/api-reference/next-config-js/headers)

---

### 5. パフォーマンス最適化の追加改善 — 低優先度

**概要:**
- 基本的なページネーションは実装済みだが、さらなる最適化が可能

**実装内容:**

1. **キャッシュ戦略の改善**
   - APIレスポンスのキャッシュ（SWR、React Query等）
   - 静的コンテンツのキャッシュ
   - ブラウザキャッシュの最適化

2. **インデックスの最適化**
   - Firestoreインデックスの設定（複合インデックス）
   - クエリパフォーマンスの測定・改善

3. **バッチ処理の改善**
   - インポート時の大量データ処理の最適化
   - チャンク処理の改善
   - 進捗表示の追加

**懸念事項:**
- Firestoreの無料枠を超えないようにする
- キャッシュの無効化タイミング
- 複雑さの増加

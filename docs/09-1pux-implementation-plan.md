# 1PUX形式のインポート/エクスポート実装計画

## 1. 要件の整理

| 項目               | 内容                                                              |
| ------------------ | ----------------------------------------------------------------- |
| **1PUX インポート** | 1Password の 1PUX 形式（JSON）をパースし、アイテムとして登録。   |
| **1PUX エクスポート** | 当サービスのアイテムを 1Password 1PUX 形式でエクスポート。       |

## 2. コンポーネント設計

| コンポーネント/層         | 役割                                           | Props / State 等                                           |
| ------------------------- | ---------------------------------------------- | ---------------------------------------------------------- |
| **1PUX インポート API**   | 1PUX JSON パース、アイテム登録                 | 入力: ファイル、グループID、出力: 登録結果                 |
| **1PUX エクスポート API** | アイテム取得、1PUX JSON 生成                   | 入力: グループID、出力: 1PUX JSON ファイル                 |
| **1PUX パーサー**        | 1PUX JSON 形式の解析                           | 入力: JSON 文字列、出力: パース済みアイテム配列             |
| **1PUX エクスポーター**   | ItemPayload を 1PUX JSON 形式に変換             | 入力: ItemPayload 配列、出力: 1PUX JSON 文字列            |
| **1PUX UI**              | ファイルアップロード、ダウンロード             | ファイル、エラーメッセージ、ダウンロード状態                |

## 3. データ設計・状態管理

- **1PUX 形式**: 1Password の標準 1PUX エクスポート形式（JSON形式、`vaults`, `items`, `fields`, `sections` 等の構造）
- **マッピング**: 1PUX のアイテム構造を当サービスの `ItemPayload` にマッピング
- **エクスポート**: `ItemPayload` を 1PUX JSON 形式に変換

## 4. 実装順序

1. **1PUX パーサーの実装**（`apps/web/src/lib/1pux/parser.ts`）
2. **1PUX エクスポーターの実装**（`apps/web/src/lib/1pux/exporter.ts`）
3. **1PUX インポート API の実装**（`/api/groups/[id]/import-1pux`）
4. **1PUX エクスポート API の実装**（`/api/groups/[id]/export-1pux`）
5. **1PUX UI の実装**（グループ詳細ページに追加）

## 5. 懸念事項・エッジケース

| 懸念                             | 対策                                                                                |
| -------------------------------- | ----------------------------------------------------------------------------------- |
| **1PUX 形式の複雑さ**            | 1PUX の標準構造を優先し、カスタムフィールドは可能な限りマッピング。                 |
| **大量データの処理**             | チャンク処理またはバックグラウンドジョブで実装。進捗をポーリングまたは SSE で返す。 |
| **ファイルサイズ制限**           | 10MB 程度の制限を設ける。                                                           |
| **エクスポート時の暗号化データ** | 復号してから 1PUX JSON に出力。                                                     |
| **ネストされた構造**             | 1PUX の `fields` と `sections` を適切に処理。                                      |

## 6. 1PUX形式の構造（参考）

1PUX形式は以下のような構造を持ちます：

```json
{
  "version": 1,
  "vaults": [
    {
      "uuid": "...",
      "name": "...",
      "items": [
        {
          "uuid": "...",
          "title": "...",
          "category": "Login",
          "fields": [
            {
              "id": "...",
              "label": "username",
              "value": "...",
              "type": "string"
            },
            {
              "id": "...",
              "label": "password",
              "value": "...",
              "type": "concealed"
            }
          ],
          "sections": [...]
        }
      ]
    }
  ]
}
```
